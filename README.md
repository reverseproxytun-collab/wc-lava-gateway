## 🇷🇺 Зачем нужен Lava.top — особенно для России

С 2022 года российские продавцы столкнулись с серьёзной проблемой: крупнейшие международные платёжные шлюзы (Stripe, PayPal) закрылись для РФ, а принимать оплату от зарубежных покупателей стало практически невозможно. **Lava.top решил эту проблему.**

Lava.top — международная платформа монетизации контента и приёма платежей, зарегистрированная на Кипре (LAVALANE LTD). Это позволяет работать с покупателями по всему миру, не зависеть от российских банковских ограничений и выводить деньги в рублях, долларах, евро и криптовалюте.

## 📦 Что такое этот плагин

**WooCommerce Lava.top Gateway (Adaptive Pro)** — это платёжный шлюз, который интегрирует вашу WooCommerce-витрину с платёжной системой Lava.top. После установки ваши покупатели смогут оплачивать заказы через Lava.top прямо на вашем сайте — без редиректов на сторонние страницы и без ручного создания контрактов.

Плагин создан с акцентом на **надёжность**, **безопасность** и **полную автоматизацию**: от создания счёта до подтверждения оплаты — всё происходит без вашего участия.

## ✨ Возможности плагина

### 💳 Платёжный процессинг

- **Разовые платежи** — стандартная оплата заказа через Lava.top с автоматическим созданием контракта по `offerId`
- **Пакетные платежи (Batch)** — корзина с несколькими товарами: для каждого позиции создаётся отдельный контракт, покупатель оплачивает их на одной странице
- **Прямые платёжные ссылки** — покупатель попадает сразу на форму оплаты, никаких лишних шагов
- **Поддержка подписок** — периодические платежи через `Lava Periodicity` (ежемесячно, еженедельно, ежегодно)
- **Идемпотентность** — повторное создание заказа не создаёт дублирующий счёт; ссылка переиспользуется в течение TTL (по умолчанию 30 минут)

### 🔄 Автоматическая сверка заказов

- **Cron-задача** (каждый час): плагин сам опрашивает Lava.top API и помечает заказы как оплаченные, если контракт получил статус `COMPLETED`
- Работает даже при потере webhook — ни один платёж не потеряется
- Диагностическая кнопка «Запустить Cron сейчас» для мгновенной проверки

### 🔗 Webhooks

- Поддержка двух режимов аутентификации: **Basic Auth** и **X-Api-Key**
- Опциональная защита по временной метке (timestamp) против атак повторного воспроизведения
- Mutex-защита от гонки при параллельных доставках
- Просмотр последних webhook-событий прямо в панели WordPress

### 🛠 Инструменты администратора

- **Health Check** — полная диагностика: SSL/TLS, совместимость валют, доступность API v1/v3, валидность API-ключа
- **Test Webhook** — отправка тестового вебхука с просмотром результата
- **Readiness Score** — показывает, какой процент ваших товаров готов к продаже через Lava.top
- **Export missing Offer IDs (CSV)** — список товаров, у которых ещё нет `offerId`
- **Export all Offer IDs (CSV)** — полный экспорт всех маппингов
- **CSV Import (двухэтапный)** — Preview → Apply с подробным отчётом об ошибках, конфликтах и обработанных строках
- **Copy Webhook URL** — одна кнопка, чтобы скопировать URL и вставить в настройки Lava.top
- **Generate Key** — безопасная генерация webhook-секрета

### 📋 Метабокс заказа

- Просмотр статуса контрактов Lava.top прямо на странице заказа в WP Admin
- Авто-обновление статуса без перезагрузки страницы
- Ссылки на платёжные URL для каждого контракта

### 🔒 Безопасность

- Все AJAX-эндпоинты защищены nonce и проверкой прав
- Escaping admin notices через `wp_kses_post`
- CSRF-защита активации лицензии (state-based pairing)
- Sanitize всех данных из webhook перед записью в заметки заказа
- Подготовленные SQL-запросы в uninstall-скриптах

### ⚙️ Совместимость

- ✅ **HPOS** (High-Performance Order Storage) — полная поддержка
- ✅ **WooCommerce Blocks** — работает в новом Checkout/Cart Block
- ✅ **WP CLI** — базовые операции через командную строку
- ✅ **WordPress 5.0+**, PHP 7.4+, WooCommerce 5.0+

## 🚀 Быстрая установка

### 1. Скачайте плагин

Скачайте `wc-lava-gateway-v2.zip` из [раздела Releases](../../releases/latest).

### 2. Установите в WordPress

**Вариант A — через админку:**
1. `Плагины → Добавить новый → Загрузить плагин`
2. Выберите `wc-lava-gateway-v2.zip`
3. Нажмите «Установить» и «Активировать»

**Вариант B — через FTP:**
```
Распакуйте архив → загрузите папку wc-lava-gateway-v2/ в /wp-content/plugins/
```

### 3. Настройте плагин

Перейдите в **WooCommerce → Настройки → Оплата → Lava.top**

1. **API Key** — вставьте публичный API-ключ из вашего аккаунта Lava.top
2. **Webhook API Key** — нажмите «Создать ключ», скопируйте значение в настройки Lava.top
3. **Режим аутентификации** — рекомендуется `X-Api-Key`
4. **Offer IDs** — откройте каждый товар и заполните поле «Lava Offer ID (UUID)»

### 4. Проверьте соединение

В разделе **Tools** запустите:
- `Health Check` — проверит API, SSL, валюты
- `Test Webhook` — отправит тестовое событие

## 🔧 Часто задаваемые вопросы

**Плагин скрывается при оформлении заказа — почему?**  
Lava.top API использует фиксированные цены по `offerId`. Шлюз автоматически скрывается, если в корзине есть купоны, скидки, налоги или стоимость доставки.

**Заказ не получает статус «Оплачен»**  
Проверьте: правильность Offer ID у товара, корректность API-ключа, соответствие webhook URL в Lava.top и в плагине, и запустите Health Check.

**Webhook работает в тестах, но timestamp показывается как отсутствующий**  
Некоторые хостинги убирают кастомные заголовки. Оставьте защиту по timestamp отключённой — это нормально.

**Могу ли я принимать подписки?**  
Да (Pro). Укажите `Lava Periodicity` в настройках товара.

**Что произойдёт после окончания триала?**  
Плагин перейдёт в режим Lite: ручная проверка оплаты, отключение фонового cron. Покупки по уже созданным контрактам продолжат работать.
